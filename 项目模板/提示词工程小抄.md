# 提示词工程小抄（学生口袋版）

> 打印出来贴在电脑旁。AI给你的好答案，90% 来自你问得好。

---

## 🥇 黄金公式：CTRA

每次问AI前，心里默念这4个字母：

- **C** Context 上下文："我在做一个XX游戏，已经完成了XX。"
- **T** Task 任务："请帮我实现XX功能。"
- **R** Requirements 要求："用HTML+JS，单文件，加中文注释。"
- **A** Acceptance 验收："运行后我应该能看到XX。"

---

## 📋 万能模板（直接抄）

### 模板1：让AI写新功能
```
我在做一个 [游戏类型] 游戏，目前已经完成了 [现状一句话描述]。

请帮我实现这个功能：[新功能一句话描述]

技术要求：
- 单个 HTML 文件，包含 CSS 和 JS
- 不用外部库
- 关键代码加中文注释

期望效果：[期望具体描述，比如"按空格键时小球往上跳50像素"]

请给我完整代码。
```

### 模板2：让AI修Bug
```
我的代码有问题：

【期望】[期望发生什么]
【实际】[实际发生了什么]
【已尝试】[你已经试过什么]

控制台报错（如果有）：
[粘贴报错]

完整代码：
[粘贴代码]

请告诉我哪里错了，并给我修改后的完整代码。
```

### 模板3：让AI解释代码
```
请用初中生能懂的话，逐行解释这段代码在做什么：

[粘贴代码]

特别说明：什么是 [你不懂的术语]？
```

### 模板4：让AI做美化
```
请保留这段代码的所有逻辑功能不变，只升级视觉：

风格：[深色霓虹 / 复古像素 / 手绘可爱 / 极简白]
主色：[紫+青 / 红+黄 / ...]
字体：[现代 / 复古 / 手写]

[粘贴代码]

给我修改后的完整代码。
```

### 模板5：让AI当玩家测试
```
请你扮演一个10岁的小朋友，第一次玩我这个游戏。
[描述游戏规则 / 贴游戏截图]
请给我3个真实的反馈：
1. 一眼看上去你想做什么？
2. 玩到第30秒你会觉得什么？
3. 你最不爽的一点是什么？
```

## ⚡ 7个让AI变聪明的技巧

1. **给例子**："我想要这样的效果：[贴一个参考链接/描述]"
2. **指定身份**："你是一个有10年经验的游戏开发者，请……"
3. **限制输出**："只给我代码不要解释" 或 "先讲思路再写代码"
4. **分步引导**："这个功能分3步：先A，再B，最后C，我们现在做A"
5. **要替代方案**："给我3种实现方式，每种说说优缺点"
6. **让它反驳自己**："这个方案最大的风险是什么？"
7. **要它先问你**："开始之前，你有什么需要我先澄清的吗？"

---

## ❌ 5个新手会犯的错

1. ❌ "帮我做个游戏" → ✅ "帮我做一个用方向键控制蓝色方块的HTML页面"
2. ❌ "不对，再来一次" → ✅ "运行后画面是黑的，应该显示方块，请检查并修改"
3. ❌ 一次让AI做5个功能 → ✅ 一次1个，跑通再下一个
4. ❌ 复制代码不看就跑 → ✅ 让AI给关键行加注释，至少要知道哪行在做什么
5. ❌ AI说能做就信 → ✅ 跑一遍验证；信代码不信承诺

## 💡 一个万能咒语
当你完全卡住、不知道怎么办时，对AI说：

> "我现在卡住了。请用三个步骤帮我：
> 1. 帮我重新整理一下我的目标
> 2. 告诉我现在最该解决的1个问题
> 3. 给我下一步的具体行动"
