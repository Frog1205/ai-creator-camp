# AI工具链全景图 · 4日创造营完整工具栈

> 一句话原则：**最少工具数量，最大覆盖能力**。
> 本表只推荐 **国内可直接访问 + 免费/有免费额度** 的工具，无需翻墙、无需信用卡。

---

## 🏆 工具栈定位

| 角色 | 工具 | 定位 |
|------|------|------|
| **主力 ①** | **秒哒AI** (miaoda.cn) | "自然语言一句话造应用" · 学生Day2核心生产工具 |
| **主力 ②** | **GLM 智谱清言** (chatglm.cn) | 通用对话主力 · 创意/文案/测试/路演全程 |
| **辅助 ①** | **Codex** (OpenAI Codex CLI) | 进阶代码调试 · 老师/高阶学生用 |
| **辅助 ②** | **WorkBuddy** | IDE 内代码辅助 · 学生Day2进阶时引入 |

为什么是这个组合？**主力解决"做出来"，辅助解决"做精细"**：
- 秒哒：让 10–16 岁零基础学生用**说人话**就能做出可玩游戏，跨过"看不懂代码"的门槛
- GLM：免费额度大、推理稳，做创意/文案/测试问答全靠它
- Codex：当学生想钻进代码改细节时，老师可用 Codex 现场示范"AI如何改代码"
- WorkBuddy：愿意接触 IDE 的进阶学生，在 VS Code/JetBrains 里得到代码补全和重构

## 🥇 主力工具详解

### ① 秒哒AI · 自然语言造应用神器

**网址**：miaoda.cn
**核心能力**：用一段中文描述，自动生成可运行的网页应用（包括小游戏）
**为什么是主力**：
- **零代码门槛**：学生不会编程也能产出
- **可直接迭代**：在产物上继续说"再加个计分""换成深色背景"即可
- **国内访问稳**：百度系，无网络问题
- **作品可分享**：生成后有独立链接，路演直接放链接

**4日营典型用法**：

| 时机 | 怎么用 |
|------|--------|
| Day2 上午 | 学生把"创意一页纸"上的描述粘贴进秒哒，10分钟出v0.1原型 |
| Day2 下午 | 在原型基础上继续追加："加3个关卡""加排行榜" |
| Day3 上午 | 根据玩家反馈追加："这里玩家觉得太难，把怪物速度调慢一半" |
| Day3 下午 | 让秒哒加音效、改配色、做开始页 |

🔑 **秒哒提示词技巧**：
```
做一个网页小游戏：
- 玩法：[2句话核心循环]
- 控制：[键盘/鼠标/触屏]
- 视觉风格：[关键词3个]
- 关键功能：[3-5个具体功能]

请直接发布，给我可访问的网页链接。
```

### ② GLM 智谱清言 · 全程对话伙伴

**网址**：chatglm.cn
**核心能力**：通用对话、推理、文档分析、创意发散
**为什么是主力**：
- **免费额度极大**：4天密集使用也不会用完
- **中文体验好**：教育场景下回答更接地气
- **支持图片/PDF**：可读学生上传的截图、读PPT资料
- **多模态生成**：能直接生成图片、PPT 大纲

**4日营典型用法**：

| 时机 | 怎么用 |
|------|--------|
| Day1 全天 | 创意发散、竞品分析、AI破冰、创意一页纸打磨 |
| Day2 卡壳时 | 秒哒生成不理想时，让GLM分析问题、改写提示词重新给秒哒 |
| Day3 反馈整理 | 把玩家反馈贴给GLM，让它分类优先级 |
| Day3 测试 | GLM 扮演 10 岁玩家给反馈 |
| Day4 路演 | 写3分钟讲稿、整理 PPT 大纲、答辩问题预演 |

## 🥈 辅助工具详解

### ③ Codex · 进阶代码助手

**形态**：OpenAI Codex CLI（终端命令行工具）/ 或 IDE 内集成
**适用场景**：
- 学生想看代码细节、改算法时
- 老师现场示范"代码层面如何与AI协作"
- 调秒哒搞不定的复杂 Bug

**4日营定位**：
- **不强推全员使用**——10岁孩子不需要
- 老师准备 1 台演示机配好 Codex，**作为示范工具**
- 12+ 高阶学生可作为奖励引入

### ④ WorkBuddy · IDE代码辅助

**适用场景**：愿意打开 VS Code 写代码的进阶学生
**定位**：
- 在 IDE 里实时代码补全、解释
- 比对话式 AI 更"贴身"
- 适合 Day3 学生想"看懂秒哒生成的代码"时引入

**4日营定位**：选修工具。**默认不教，学生主动问起再讲**。

## ⚠️ 常见问题预案

| 问题 | 应对 |
|------|------|
| 秒哒生成的应用不符合预期 | 让 GLM 帮学生重写提示词，再丢回秒哒 |
| 秒哒额度用完 | 用 GLM 直接生成 HTML 代码作为备份方案 |
| 学生想"看代码" | 老师切换到 Codex 现场示范代码改动 |
| 学生过度依赖秒哒"一键出货" | 加规则：**每天至少1个功能学生用自己的提示词反复打磨3次以上才算通过** |
| 网络抖动 | 准备3个示范案例的本地 HTML 文件兜底 |

## 🎓 工具引入节奏建议（重要！）

| 时机 | 引入哪个工具 | 为什么这时引入 |
|------|--------------|---------------|
| Day1 上午 | **只用 GLM 对话** | 一次只学一个工具，建立信心 |
| Day1 下午 | 加 **xmind/思维导图** | 创意结构化时引入 |
| Day2 上午 | 引入 **秒哒AI** ← 当天的主角 | 学生开始真正"造游戏"的时刻 |
| Day2 下午 | 演示 **Codex** （选讲） | 让学生看到"代码层面的AI协作" |
| Day3 上午 | 加 **即梦AI / SVG生成（GLM）** | 美化阶段才需要美术 |
| Day3 下午 | 加 **Web Audio音效 / 剪映录屏** | 资产丰富阶段 |
| Day4 上午 | 加 **WPS AI / GLM出PPT** | 路演冲刺一次教完 |
| Day4 下午 | 加 **剪映 + 海螺AI配音** | 最终视频包装 |

✨ **核心原则**：**不要在Day1把所有工具都讲完**。学生注意力是宝贵资源，**用到时才引入**，每个工具都有自己的"高光时刻"。

## 🆚 为什么不推荐这些"看起来很热"的工具

| 工具 | 不推的原因 |
|------|----------|
| Claude / ChatGPT / Gemini | 国内访问不便，4日营节奏被网络打断风险高 |
| Cursor / Trae（默认） | IDE 学习曲线陡，10岁学生上手成本高 |
| Midjourney / SD | 翻墙/付费/学习成本三重门槛 |
| Notion AI | 国内未本地化，登录都困难 |

**不是这些工具不好，是不适合这个营的目标人群和时间预算。**
