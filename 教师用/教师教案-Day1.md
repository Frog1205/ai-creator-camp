# 教师教案 · Day 1：启动·探索

## 📌 教学目标
1. **认知**：学生能用自己的话说出"AI是协作者不是答案机"
2. **技能**：学生能通过提问让AI产出有用内容（提示词意识初步建立）
3. **情感**：学生对"我能做出一个游戏"建立信心
4. **产出**：每组一份《创意一页纸》

## 🧰 课前准备
- [ ] 每组1台电脑，预装Chrome、记事本/VS Code
- [ ] 每人一份《学生手册-Day1.md》打印版
- [ ] 投影仪 + 教师演示账号（提前注册好AI助手）
- [ ] A3纸/便利贴/马克笔（创意一页纸用）
- [ ] **预先测试**：所有电脑都能正常访问选定的AI助手

## ⏱ 课堂节奏

### 09:00–09:30 开营仪式（30')
- **开场1分钟视频**：放一段"小学生用AI做游戏"的真实案例（如YouTube/B站可找到）
- **破冰**：每人30秒自我介绍 + "你玩过最上头的游戏是？"
- **公布终点**：明确"第4天每组要做一个3分钟路演"——**终点要在第1分钟就立起来**

### 09:30–10:30 任务1：AI破冰（60'）
**教师演示（10')**：现场用AI生成红色方块HTML，**故意**让AI出一次错，然后纠正它，让学生看到"AI会错、人要判断"。

**学生操作（40')**：自由探索，老师巡场。

**集体分享（10')**：抽3名学生分享"AI最让你意外的回答"。

⚠️ **易错点**：
- 学生会问AI"你能做什么"——这种宽泛问题AI答不好。引导：**问具体的事**。
- 部分学生会怕"问错"——告诉他们 **AI是问不坏的，多问多错**。

### 10:30–11:30 任务2：好游戏拆解（60'）
- 教师先做1个示范（推荐《合成大西瓜》，学生熟悉）
- 小组讨论 + 全班归纳：把"3个让游戏好玩的共同点"写在黑板上
- **典型答案**：及时反馈、难度曲线、收集感/成就感

### 11:30–12:00 任务3：头脑风暴（30'）
- 强调"**先求多、再求精**"——不要写到第3个就停
- 巡场鼓励"奇怪的组合"——越怪越有戏

### 14:00–15:30 任务4：AI验证创意（90'）
**关键引导**：当AI说"难度5星"时，**不要让学生换创意**，先教他们**砍功能**。
- 示范：原创意"3D开放世界RPG" → 砍到 "2D俯视视角 + 1张地图 + 1个敌人"
- 这一步是教学**最高价值**的瞬间：**项目管理思维的种子**

### 15:30–16:30 任务5：创意一页纸（60'）
- 提供模板（见《项目模板/创意一页纸模板.md》）
- 要求**手写或电脑都行**，关键是"5米外能看清"——这是为明天演示做铺垫

### 16:30–17:00 互评 + 复盘（30'）
- 走廊画廊式：所有创意贴墙，每人发3个小红点贴在最想玩的3个上
- 复盘问题（口头）：今天最大的"啊哈时刻"？

## 🎯 引导话术参考库
- 学生问 "AI写的代码我看不懂怎么办？" → "看不懂没关系，先问AI'用小学生能懂的话解释每一行'。"
- 学生说 "AI说我这个做不出来。" → "AI说的'做不出来'通常意思是'4天里做不完整版'。砍功能，留核心。"
- 学生说 "我想做和别人一样的（比如都做贪吃蛇）。" → "可以，但加一个'只有你们组才有的twist'。"

## 🚨 风险预案
| 风险 | 应对 |
|------|------|
| AI助手访问慢/抽风 | 备用方案：至少2个不同的AI助手 |
| 学生卡在创意阶段 | 准备15个"种子创意"卡片，抽签救场 |
| 个别学生跑得太快 | 给延伸任务："帮邻桌问3个好问题给AI" |
| 个别学生进度太慢 | 让队友"代问"AI，先确保不被落下 |

## 📊 Day1 形成性评价点
观察并记录（每生1–5分）：
- 提问质量（是否在用具体提示词）
- 协作参与（是否在和队友讨论）
- 反思深度（复盘是否有具体细节）

## 🏠 教师当晚准备

2. 准备Day2的"通用代码模板"应急包（万一某组AI完全跑不出来，可用模板兜底）
