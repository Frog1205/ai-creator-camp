# 案例拆解教学卡 · 如何用3个示范作品上好这一课

> 参考 AI探索营 V2 经验：示范案例不是"答案"，是**讲解游戏五要素和提示词工程的活教材**。
> 教学顺序建议：先让学生玩 → 再拆游戏五要素 → 最后看背后的提示词。

---

## 🎮 案例A · 记忆翻翻乐（休闲解谜赛道）

**讲解时机**：Day 1 任务2 "好游戏拆解" 环节，作为示范样板。

### 拆解卡（板书或PPT）

| 游戏五要素 | 这个游戏怎么做的 |
|------------|------------------|
| **目标** | 用最少步数把8对图案全部配对消除 |
| **规则** | 一次翻2张，相同消除，不同翻回去；不能翻已消除的 |
| **反馈** | 翻牌3D动画 + 配对成功变绿色弹跳 + 实时计步/计时 |
| **挑战** | 记忆容量（位置）+ 决策速度（计时压力）|
| **上瘾点** | "再玩一把刷新最少步数纪录"——可量化的自我超越 |

### 拆功能（演示给学生看怎么分步做）
1. 画一个4×4网格 ←—— **先做这一步！**
2. 加一个数据数组：16张牌，8对emoji，洗牌
3. 点击翻牌动画
4. 配对逻辑：连续2张相同→消除
5. 计步 + 计时 + 通关界面

⭐ **关键教学点**：学生最常犯的错——**想一次性让AI生成全部**。这个案例可以打开源代码，让学生看：**每段代码对应哪个功能**。

### 背后的提示词（展示给学生）
```
做一个4×4记忆翻牌游戏，单个HTML文件：
- 8对emoji图案，开局打乱
- 点击翻牌3D动画
- 翻对2张消除，翻错翻回去
- 计步、计时、通关弹窗
- 深色背景 + 橙红渐变主题
关键代码加中文注释。给我完整代码。
```

## 🚀 案例B · 星际接接乐（动作冒险赛道）

**讲解时机**：Day 2 任务2 "第一版代码"环节，演示Canvas游戏标准结构。

### 拆解卡

| 游戏五要素 | 这个游戏怎么做的 |
|------------|------------------|
| **目标** | 接到尽可能多的星星，避开陨石不死 |
| **规则** | 飞船左右移动；接星星+10；撞陨石-1命；3条命用完结束 |
| **反馈** | 飞船尾焰动效、星星金色发光、陨石灼热阴影、分数飙升 |
| **挑战** | 每100分提升关卡，陨石下落变快变多 |
| **上瘾点** | "再来一次能冲多少分"——分数的边际刺激 |

### 这个案例展示了Canvas游戏的5个核心模块（板书）
1. **游戏循环**：`requestAnimationFrame(update)`
2. **状态管理**：`ship`/`items`/`score` 等变量
3. **绘制**：`ctx.fillStyle` + `ctx.beginPath` 等
4. **输入**：键盘 `keydown/keyup` + 触屏 `touchmove`
5. **碰撞检测**：`Math.abs(dx) < r1+r2`

🎯 **教学法**：让学生**关闭代码**先玩2分钟，再打开代码**找出**上面5个模块分别在哪几行——这比讲解更有效。

### 背后的提示词（学生可以学这个结构）
```
做一个HTML5 Canvas动作小游戏：
- 玩家控制飞船（方向键 + 触屏均可）左右移动
- 星星从上方掉落，接到+10分
- 陨石也从上方掉落，撞到-1命，3条命用完结束
- 分数每涨100，关卡+1，陨石变快变多
- 视觉：深紫色太空背景，飞船尾焰，星星金色发光
单文件HTML，约300行以内。给我完整代码。
```

## 📜 案例C · 成语接龙王（知识策略赛道）

**讲解时机**：Day 3 任务1 "迭代+美化"环节，启发"学科融合"思路。

### 拆解卡

| 游戏五要素 | 这个游戏怎么做的 |
|------------|------------------|
| **目标** | 30秒内接尽量多的成语，刷新积分纪录 |
| **规则** | 用前一个成语的末字开头（同音字也可以） |
| **反馈** | 接对绿色提示+连击×N、错误红色提示、积分实时跳 |
| **挑战** | 倒计时压力 + 词库要求 + 不能重复 |
| **上瘾点** | 连击系统 + 时间奖励——越紧张越想再试一次 |

### 这个案例的"教学价值"

它是**最简单的代码**，但**最有创意的设计**——这正是要让学生看到的：
> **AI的代码能力是相对均等的，差别在于人的创意。**

### 启发学生用 AI 做"学科+游戏"
- 数学版：算式接龙、24点
- 英文版：单词最后一个字母 → 下一个单词
- 历史版：朝代年表问答
- 化学版：元素周期表配对

⭐ **课堂活动**：让学生现场想3个"X+游戏"，并让AI快速生成一个玩具版试一试（15分钟挑战）。

### 背后的提示词
```
做一个成语接龙网页游戏：
- 30秒倒计时
- 内置约30条常用成语词库（{首字:[成语列表]}结构）
- 用户输入后判断：是否四字、是否首字匹配（含同音字）、是否已用过
- 连击系统：连击×N加分；连击3次奖励+3秒
- 古风UI：卷轴金色 + 楷体字
单文件HTML。给我完整代码。
```

## 🎓 三个案例的共同教学要点

### 1. 难度递进（学生可以模仿这个曲线）
- 案例A：纯JS逻辑（无Canvas）——零基础起步
- 案例B：Canvas + 动画循环——技术深一层
- 案例C：游戏机制 + 数据结构（词库）——逻辑深一层

### 2. 都遵守 "200–500行 / 单文件 / 无外部库"
让学生看到：**好游戏不一定是大游戏**。

### 3. 都有"再玩一次"的钩子
- A：刷新最少步数
- B：刷新最高分
- C：刷新连击纪录

**教学话术**：
> "一个让玩家想'再玩一次'的游戏，已经赢了一半。你的游戏，那个'再玩一次'的理由是什么？"

## 📋 教学使用建议

| 时机 | 怎么用案例 |
|------|------------|
| **开营第1天** | 让学生玩A，作为"麻雀虽小五脏俱全"的展示 |
| **Day1 创意阶段** | 用三个案例演示"赛道选择"——你想做休闲、动作还是知识？ |
| **Day2 卡壳时** | 打开对应赛道案例，让学生**看代码结构** + **抄代码模式** |
| **Day3 反馈整理** | 用案例对比："为什么人家的提示这么清楚？" |
| **Day4 路演前** | 让学生玩别组的作品前，先回顾这3个案例的标准 |

⚠️ **避免**：把案例当"答案"分发。学生抄了就没意思了。**先讲思路，再亮代码**。
