# 教师教案 · Day 4：路演·答辩

## 📌 教学目标
1. **认知**：理解"作品交付"和"作品制作"是两种不同能力
2. **技能**：能在3分钟内清晰讲述项目故事
3. **情感**：在公开场合获得成就感 + 学会接受不同意见
4. **产出**：完成路演 + 复盘表 + 颁奖

## ⏱ 课堂节奏

### 09:00–10:00 最后冲刺（60'）
**严格规则**：
- 09:30 之后**禁止改代码**（再改容易翻车）
- 09:30–10:00 只允许：PPT调整 + 排练 + 录视频备份
- 老师在黑板倒计时：30分钟 / 15分钟 / 5分钟

### 10:00–11:00 排练（60'）
**两轮排练法**：
- 第一轮：每组在自己位置讲一遍，队友互相挑刺（5分钟讲+5分钟反馈，10分钟/组）
- 第二轮：上台试讲（用真投影 + 真话筒，每组5分钟）

老师只盯三件事：
1. 时间是否在3分钟内
2. 演示画面是否清晰可见
3. 开头30秒有没有抓住注意力

### 11:00–15:00 路演（含午休）

**上半场（11:00–12:00）**：3–4组
**下半场（14:00–15:00）**：3–4组

📋 **每组流程**（每组8分钟）：
- 路演：3分钟
- 现场玩：1分钟（评委/同学玩一下）
- 答辩：3分钟（评委2个问题）
- 评分：1分钟（评委填表）

🎤 **主持词模板**（老师/主持人）：
> "下一组带来的是《XXX》，掌声有请！"
> （演示后）"谢谢XX组。评委有什么问题？"
> （答辩后）"请评委们打分，我们继续。"

### 15:00–16:00 互评 + 评委点评（60'）
- **学生互评**：每组用《评价量规.md》给所有其他组打分（不能给自己打）
- **评委点评**：每位评委选**1–2组**做3分钟深度点评——**只讲优点和具体建议**，**不做横向比较**
- 老师汇总分数

### 16:00–17:00 复盘 + 颁奖（60'）
**复盘**（30'）：
- 每人独立填《项目模板/项目复盘表.md》
- 全班围圈分享：每人**1句话**——"这4天我最重要的发现是____"

**颁奖**（30'）：
- 5–6个不同类别奖项（创意/人气/AI协作/迭代/潜力/团队）
- **保证每组至少拿一个奖**——这是教育现场，不是淘汰赛
- 颁奖词由老师准备，**讲出每组的"独特之处"**

### 17:00–17:30 结营仪式（30'）
- 集体合影
- 发结营证书（提前印好，盖章）
- 给学生留一道"作业"：**回家把游戏发给3个朋友玩，记录他们的反应**——让学习延续到营外

## 🎯 评委邀请话术（如有外部评委）
> "我们邀请您不是来挑刺的，是来'认真当一次玩家'的。请用具体的话表扬，用建设性的话建议。学生们花了4天，我们花他们一上午来肯定他们的勇气。"

## 🚨 风险预案
| 风险 | 应对 |
|------|------|
| 某组代码现场跑不起来 | 切到Demo视频（昨天录的） |
| 某学生紧张说不出话 | 队友补位 + 老师插话："我们听XX讲讲他做的部分。" |
| 评委过于挑剔 | 主持人接话："这是很好的建议，下一版可以考虑。"——主动护场 |
| 某组明显落后 | 颁"最敢于挑战奖"——把他们的故事讲出来 |

## 🏆 奖项设计（建议）
| 奖项 | 评选标准 |
|------|---------|
| 最佳创意奖 | 想法最新颖 |
| 最佳人气奖 | 互评最想玩 |
| 最强AI协作奖 | 提示词工程做得最深入 |
| 最快迭代奖 | Day3 反馈→改进 闭环最好 |
| 最强讲故事奖 | 路演最打动人 |
| 最具潜力奖 | 给"还没完全做完但方向极好"的组 |
